## 功能特性

| 功能 | 说明 |
|------|------|
| 历史回测 | 日线级别事件驱动回测，真实模拟 A 股交易成本 |
| 盘前分析 | 启动时自动分析每只股票所处区间（买入/卖出/中性），彩色弹窗提醒 |
| 信号监控 | AKShare 轮询，盘中每 15 分钟检查，红/绿色醒目提醒 + 声音 + 弹窗 |
| 模拟盘 | 内存撮合，假钱下单，逻辑与实盘完全一致 |
| 实盘接入 | 掘金量化 SDK，30 分钟 K 线触发，真实委托 |
| 双数据源 | 东方财富 + 腾讯财经自动切换，解决限流问题 |
| 一键切换 | `config.py` 中一行配置切换模拟盘/实盘 |
| 技术指标 | MA / EMA / MACD / 布林带 / RSI / ATR 自动计算 |
| 多信号评分 | 均线金叉、MACD 金叉、RSI 超卖反弹多维度评分，综合打分决策 |
| 分级止盈 | 6%/10%/25% 三档分批止盈（2024优化后关闭，让利润跑）|
| 移动止盈 | 20% 移动止盈跟踪最高价，保护浮盈 |
| 全面风控 | 单票仓位上限、总仓位上限、最大回撤清仓、止损 |
| T+1 保护 | 当日买入当日不可卖出（回测和模拟盘均有效）|
| 绩效报告 | 夏普比率、卡玛比率、最大回撤、胜率等，自动生成图表 |
| 本地缓存 | 历史数据缓存到磁盘（`.pkl`），避免重复下载 |

## 快速开始

### 第一步：安装依赖

```bash
cd F:\quant
pip install -r requirements.txt
```

> **注意：** `gm`（掘金 SDK）仅实盘需要，如果只做回测和信号监控可以跳过：
> ```bash
> pip install akshare pandas numpy
> ```
>
> **推荐安装弹窗通知库**（信号监控时弹出 Windows Toast 通知）：
> ```bash
> pip install win10toast
> ```
> 不安装也能运行，系统会降级使用 `ctypes.MessageBoxW` 阻塞式弹窗。

### 第二步：运行回测

无需任何账号，直接运行：

```bash
py main_backtest.py
```

同时生成：
- 交易记录：`reports/trades_MACross_YYYYMMDD.csv`
- 回测图表：`reports/MACross_YYYYMMDD.png`（需安装 matplotlib）

### 第三步：信号监控（推荐，不需要掘金账号）

不需要任何额外配置，直接运行：

```bash
py main_signal.py
```

系统会：
- **启动时立即分析**每只股票所处区间（买入/卖出/中性），弹窗提醒

- 有买卖信号时**红色/绿色醒目提醒** + 声音 + 弹窗
- 非交易时间自动休眠，盘前自动唤醒
- **不自动下单**，你在自己的交易软件中手动操作
- 首次运行会自动下载历史数据并缓存，后续直接读缓存

当策略检测到买卖信号时，控制台会显示**红色/绿色醒目提示**：


确认模拟盘稳定后再切换实盘：`TRADE_MODE = "live"`。

## 策略介绍

### 主策略：双均线 + MACD（MA Cross）

**文件：** `strategy/ma_cross.py`

**买入逻辑（满足任一 + 过滤条件）：**
- 信号A：MA5 上穿 MA30（均线金叉）
- 信号B：MACD 柱由负转正（MACD金叉）
- 过滤：RSI < 80、成交量 > 20日均量50%、2日冷却期
- 注：趋势过滤（MA_TREND）在 2024 优化中**关闭**（设为0），因为它阻碍了捕捉 2024 下半年反弹

**卖出逻辑（多层保护）：**
- MA5 下穿 MA30（均线死叉）→ 全仓卖出
- MACD 死叉 + RSI > 90 → 全仓卖出
- RSI > 90 极端超买 + 盈利 → 减仓 50%
- 止损：亏损 12% 强制卖出（R6最优，14股更紧控制）
- 移动止盈：从最高价回撤 20%（实测几乎不触发）
- **分级止盈已关闭**（R1-R6 一致结论：让利润跑！）

**关键参数：** MA5/MA30（快慢均线，R6最优），无趋势过滤，高仓位（90%），分级止盈关闭。

---

### 观察策略：MACD

**文件：** `strategy/macd.py`

- MACD 柱由负转正（DIF 上穿 DEA）→ 打印"买入观察"日志
- MACD 柱由正转负（DIF 下穿 DEA）→ 打印"卖出观察"日志
- **不会下单**，仅供参考。

---

### 观察策略：布林带（Bollinger Bands）

**文件：** `strategy/boll.py`

- 价格触及下轨 → 打印"超卖观察"日志
- 价格回归中轨以上 → 打印"回归观察"日志
- **不会下单**，仅供参考。

## 风控规则

风控由 `risk/manager.py` 统一管理，在每次策略信号执行前介入：

| 规则 | 触发条件 | 处理方式 |
|------|---------|---------|
| 单票仓位上限 | 买入后该股票市值 > 总资产 35% | 削减买入数量或拒绝 |
| 总仓位上限 | 所有持仓市值 > 总资产 95% | 拒绝买入信号 |
| 现金保留 | 买入后现金 < 总资产 5% | 拒绝买入信号 |
| 单票止损 | 持仓亏损 >= 12% | 生成强制卖出信号 |
| 移动止盈 | 持仓从最高价回撤 >= 20% | 生成强制卖出信号 |
| 最大回撤清仓 | 总资产从峰值回撤 >= 50% | 生成全仓清仓信号 |

**修复：** 最大回撤清仓后会重置峰值资产，避免清仓后永久锁定无法再次开仓。

## 实盘使用

### 方式一：信号监控 + 手动下单（推荐）

不需要掘金账号，直接运行：

```bash
py main_signal.py
```

- 使用 AKShare 免费数据源（东方财富 + 腾讯双源），每 15 分钟轮询一次
- 有信号时红色/绿色提醒，你在自己的交易软件（同花顺、东方财富等）手动操作
- 交易时间外自动休眠，盘前自动唤醒
- 首次运行自动下载历史数据并缓存，后续直接读缓存（秒级启动）
- `Ctrl+C` 退出

### 方式二：掘金自动下单（需要掘金账号）

#### 获取掘金配置

1. **GM_TOKEN**：登录 [myquant.cn](https://www.myquant.cn) -> 个人中心 -> 系统设置 -> Token
2. **ACCOUNT_ID**：打开掘金终端 -> 实盘交易 -> 账户列表 -> 账户 ID
3. **STRATEGY_ID**：登录网站 -> 我的策略 -> 新建策略 -> 获取策略 ID

#### 掘金运行流程


## 数据源说明

### 双数据源自动切换

`data/provider.py` 实现了双数据源自动切换机制：

| 数据源 | AKShare 接口 | 优点 | 缺点 |
|--------|-------------|------|------|
| 东方财富 | `stock_zh_a_hist` | 速度快，支持日期过滤 | 限流严重，频繁调用会被封 IP |
| 腾讯财经 | `stock_zh_a_daily` | **不限流**，数据完整 | 返回全量数据需手动裁剪 |

**工作流程：**
1. 先尝试东方财富（最多重试 2 次，间隔 3 秒）
2. 如果东方财富失败（限流/超时），自动切换到腾讯财经
3. 腾讯财经作为兜底，确保数据一定能拿到

**实时价格：**
- 优先东方财富 `stock_zh_a_spot_em`（一次拿全市场快照）
- 失败后用腾讯最近一条日线收盘价作为近似实时价

## 常见问题

**Q：运行报错 `ImportError: No module named 'akshare'`**

```bash
pip install akshare
```

**Q：东方财富限流了怎么办？**

不用管。系统会自动切换到腾讯财经数据源，完全无感。日志中会看到：
```
[东方财富] 600519 第1次失败: Remote end closed connection
东方财富不可用，切换到腾讯数据源...
[腾讯] 下载成功 600519：277 条
```

**Q：首次运行 main_signal.py 很慢？**

首次运行需要下载历史数据（约 1~2 分钟），数据会缓存到 `data_cache/` 目录。后续运行直接读缓存，秒级启动。

**Q：运行实盘报错 `掘金 SDK 未安装`**

信号监控模式（`main_signal.py`）不需要掘金 SDK。只有使用 `main_live.py` 自动下单才需要：
```bash
pip install gm
```

**Q：如何只监控部分股票？**

在 `config.py` 中修改，例如只监控茅台：

```python
SYMBOLS_RAW = ["600519"]
SYMBOLS_GM  = ["SHSE.600519"]
SYMBOL_NAMES = {"600519": "贵州茅台"}
```

> 注意：减少股票数量后，建议重新优化参数（POSITION_PCT 和 MAX_POSITION_PCT 需要调整）。

**Q：T+1 是什么意思？**

A 股规定当天买入的股票，当天不能卖出，要等到下一个交易日才能卖。系统在回测和模拟盘中均做了 T+1 保护。

**Q：如何清除历史数据缓存？**

删除 `data_cache/` 目录下的所有 `.pkl` 文件，或在 Python 中调用：

```python
from data.manager import DataManager
dm = DataManager()
dm.clear_cache()          # 清除全部缓存
dm.clear_cache("600519")  # 只清除茅台的缓存
```

**Q：回测结果和之前不一样了？**

数据源切换到腾讯后，历史数据可能有微小差异（不同数据源的复权计算方式略有不同），属正常现象。
