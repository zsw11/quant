## 项目简介

本系统专为 A 股市场设计，以「**只改一个文件**」为核心理念——所有参数集中在 `config.py`，用户无需修改其他任何文件，即可完成从回测验证到实盘上线的全流程。

**当前监控标的：**
- 贵州茅台（600519 · 上交所）
- 宁德时代（300750 · 深交所）

**数据来源：**
- 回测：AKShare（免费，日线历史数据，自动缓存到本地）
- 实盘：掘金量化 SDK（30 分钟 K 线，实时推送）

## 功能特性

| 功能 | 说明 |
|------|------|
| 历史回测 | 日线级别事件驱动回测，真实模拟 A 股交易成本 |
| 模拟盘 | 内存撮合，假钱下单，逻辑与实盘完全一致 |
| 实盘接入 | 掘金量化 SDK，30 分钟 K 线触发，真实委托 |
| 一键切换 | `config.py` 中一行配置切换模拟盘/实盘 |
| 技术指标 | MA / EMA / MACD / 布林带 / RSI / ATR 自动计算 |
| 全面风控 | 单票仓位上限、总仓位上限、最大回撤清仓、止损 |
| T+1 保护 | 当日买入当日不可卖出（回测和模拟盘均有效）|
| 绩效报告 | 夏普比率、卡玛比率、最大回撤、胜率等，自动生成图表 |
| 本地缓存 | 历史数据缓存到磁盘，避免重复下载 |

## 快速开始

### 第一步：安装依赖

```bash
cd F:\quant
pip install -r requirements.txt
```

> **注意：** `gm`（掘金 SDK）仅实盘需要，如果只做回测可以跳过：
> ```bash
> pip install akshare pandas numpy matplotlib
> ```

### 第二步：运行回测

无需任何账号，直接运行：

```bash
python main_backtest.py
```

控制台会打印：

```
============================================================
  回测结果报告 ── MACross
============================================================
  回测区间  : 20220101 ~ 20241231
  初始资金  :      1,000,000 元
  最终资产  :      1,234,567.89 元

  年化收益率:          10.12 %
  最大回撤  :         -14.32 %
  夏普比率  :          0.8821
  卡玛比率  :          0.7072
  年化波动率:          18.50 %
  总交易次数:              42
  胜率      :          61.5 %
  累计手续费:       3,456.78 元

```

同时生成：
- 回测图表：`reports/MACross_YYYYMMDD.png`
- 交易记录：`reports/trades_MACross_YYYYMMDD.csv`

### 第三步：模拟盘验证（需要掘金账号）

在 `config.py` 中填写掘金配置：

```python
GM_TOKEN    = "你的掘金Token"       # myquant.cn → 个人中心 → Token
ACCOUNT_ID  = "你的账户ID"          # 掘金终端 → 实盘交易 → 账户ID
STRATEGY_ID = "你的策略ID"          # myquant.cn → 我的策略 → 策略ID
TRADE_MODE  = "paper"               # 先用模拟盘，假钱验证
```

然后运行：

```bash
python main_live.py
```

### 第四步：切换实盘

**确认模拟盘运行稳定后**，将 `config.py` 中的一行改为：

```python
TRADE_MODE = "live"   # 真实账户，真实下单！
```

重新运行：

```bash
python main_live.py
```

```python
# ---- 监控股票 ----

# ---- 资金 ----
INITIAL_CAPITAL = 1_000_000   # 初始资金（回测和模拟盘使用）

# ---- 双均线参数 ----

MA_SLOW      = 20    # 长期均线（20根30分钟K线）

# ---- 风控参数 ----
MAX_POSITION_PCT   = 0.30   # 单票最大仓位30%
MAX_TOTAL_POS_PCT  = 0.95   # 总仓位上限95%
MAX_DRAWDOWN_LIMIT = 0.15   # 最大回撤15%触发全仓清仓
STOP_LOSS_PCT      = 0.05   # 单票止损5%
MIN_CASH_RESERVE   = 0.05   # 最低现金保留5%

# ---- 回测时间范围 ----
BACKTEST_START = "20220101"
BACKTEST_END   = "20241231"

# ---- 交易成本 ----
COMMISSION     = 0.0003   # 佣金万3
STAMP_DUTY     = 0.001    # 印花税千1（仅卖出）
SLIPPAGE       = 0.002    # 滑点2‰
MIN_COMMISSION = 5.0      # 最低佣金5元

# ---- 实盘K线 ----
BAR_FREQUENCY = "30m"   # 30分钟K线
BAR_COUNT     = 60      # 每次获取60根（足够计算MA20）
```

## 策略介绍

### 主策略：双均线（MA Cross）

**文件：** `strategy/ma_cross.py`

**逻辑：**
- 金叉（MA5 上穿 MA20）→ **买入**，使用可用资金的 90%，按 100 股取整
- 死叉（MA5 下穿 MA20）→ **卖出**全部持仓
- 未持仓时不卖，已持仓时不重复买（防止反复开仓）

**特点：** 趋势跟踪策略，适合单边上涨行情，震荡市会被反复打脸。

---

### 观察策略：MACD

**文件：** `strategy/macd.py`

**逻辑：**
- MACD 柱由负转正（DIF 上穿 DEA）→ 打印"买入观察"日志
- MACD 柱由正转负（DIF 下穿 DEA）→ 打印"卖出观察"日志

**重要：** `quantity = 0`，**不会下单**，仅供参考。可以用来判断主策略信号是否得到 MACD 的确认。

---

### 观察策略：布林带（Bollinger Bands）

**文件：** `strategy/boll.py`

**逻辑：**
- 价格触及下轨 → 打印"超卖观察"日志
- 价格回归中轨以上 → 打印"回归观察"日志

**重要：** `quantity = 0`，**不会下单**，仅供参考。布林带适合震荡市，与双均线形成互补。

## 风控规则

风控由 `risk/manager.py` 统一管理，在每次策略信号执行前介入：

| 规则 | 触发条件 | 处理方式 |
|------|---------|---------|
| 单票仓位上限 | 买入后该股票市值 > 总资产 30% | 拒绝该买入信号 |
| 总仓位上限 | 所有持仓市值 > 总资产 95% | 拒绝买入信号 |
| 现金保留 | 买入后现金 < 总资产 5% | 拒绝买入信号 |
| 单票止损 | 持仓亏损 ≥ 5% | 生成强制卖出信号 |
| 最大回撤清仓 | 总资产从峰值回撤 ≥ 15% | 生成全仓清仓信号，丢弃其他信号 |

## 实盘使用

### 获取掘金配置

1. **GM_TOKEN**：登录 [myquant.cn](https://www.myquant.cn) → 个人中心 → 系统设置 → Token
2. **ACCOUNT_ID**：打开掘金终端 → 实盘交易 → 账户列表 → 账户 ID
3. **STRATEGY_ID**：登录网站 → 我的策略 → 新建策略 → 获取策略 ID

### 实盘运行流程

```
第一步：TRADE_MODE = "paper"  →  python main_live.py  →  观察日志是否正常
第二步：确认逻辑无误后        →  TRADE_MODE = "live"   →  python main_live.py
```

### 实盘日志位置

运行后日志自动保存到：

```
logs/live_YYYYMMDD_HHMMSS.log
```

### 策略在实盘中的执行频率

掘金订阅了 30 分钟 K 线，每根 K 线收盘后触发一次 `on_bar`：

```
09:30 开盘
10:00  ← 第一根30分钟K线结束，触发策略
10:30  ← 第二根
...
14:30  ← 最后一根
15:00 收盘（收盘竞价阶段的K线也会触发）
```

A 股每天交易时间内共触发约 **8 次**策略检查。

## 交易成本

系统严格模拟 A 股真实交易成本：

| 成本项 | 标准 | 备注 |
|--------|------|------|
| 佣金 | 万 3（0.03%）| 买入和卖出均收取 |
| 印花税 | 千 1（0.1%）| **仅卖出收取** |
| 最低佣金 | 5 元/笔 | 交易金额小时按此计算 |
| 滑点 | 2‰（0.2%）| 买入价格略高，卖出价格略低 |

**示例：** 买入贵州茅台 100 股，股价 1800 元

```
成交金额 = 1800 × 100 = 180,000 元
佣金     = max(180,000 × 0.03%, 5) = 54 元
滑点     = 1800 × 0.2% = 3.6 元/股（实际成交价 1803.6 元）
合计成本 = 180,360 + 54 = 180,414 元
```

## 常见问题

**Q：运行回测报错 `ImportError: No module named 'akshare'`**

```bash
pip install akshare
```

**Q：AKShare 下载数据很慢或超时**

数据第一次下载会联网，之后会缓存到 `data_cache/` 目录，下次直接读取本地缓存，速度很快。网络不好时可以多试几次。

**Q：运行实盘报错 `掘金 SDK 未安装`**

```bash
pip install gm
```

**Q：掘金 SDK 安装失败**

掘金 SDK 需要配合掘金终端使用，请先从 [myquant.cn](https://www.myquant.cn) 下载并安装掘金终端，再安装 Python SDK。

**Q：回测结果看起来很好，实盘为什么亏钱？**

回测存在以下局限性，属正常现象：
- 回测使用日线数据，实盘使用 30 分钟 K 线，信号触发点不同
- 回测无法模拟真实的流动性冲击（大单导致的价格偏移）
- 过去的规律不一定在未来重复

建议先用模拟盘（`TRADE_MODE = "paper"`）运行至少 1-3 个月，确认策略在实时行情中表现符合预期再切换实盘。

**Q：如何只监控茅台，不监控宁德时代？**

在 `config.py` 中修改：

```python
SYMBOLS_RAW = ["600519"]
SYMBOLS_GM  = ["SHSE.600519"]
```

**Q：T+1 是什么意思？**

A 股规定当天买入的股票，当天不能卖出，要等到下一个交易日才能卖。系统在回测和模拟盘中均做了 T+1 保护，实盘由掘金风控系统保障。

**Q：如何清除历史数据缓存？**

删除 `data_cache/` 目录下的所有 `.pkl` 文件，或在 Python 中调用：

```python
from data.manager import DataManager
dm = DataManager()
dm.clear_cache()          # 清除全部缓存
dm.clear_cache("600519")  # 只清除茅台的缓存
```
